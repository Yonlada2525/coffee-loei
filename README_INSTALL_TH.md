# Coffee Loei DB

เว็บแอป React + Node.js + MySQL สำหรับระบบฐานข้อมูลพื้นที่ปลูกกาแฟจังหวัดเลย

## โครงสร้าง

```text
coffee_loei_db/
├── frontend/   React + Vite + Tailwind CSS
├── backend/    Node.js + Express API
└── database/   coffee_loei_db.sql
```

## ขอบเขตระบบที่รองรับ

### ผู้ดูแลระบบ
- เข้าสู่ระบบ
- เพิ่ม แก้ไข ลบ และตรวจสอบข้อมูลเจ้าของสวน
- เพิ่ม แก้ไข ลบข้อมูลสวนกาแฟและตำแหน่งบนแผนที่
- อัปโหลดรูปภาพ/สื่อประกอบ
- เพิ่ม แก้ไข ลบข้อมูลกาแฟ
- ดูรายงานจำนวนสวนและสถิติผลผลิตทั้งหมด
- ดูกราฟ แผนภูมิ และสรุปทางสถิติ
- ส่งออกข้อมูลสวนเป็น Excel, CSV และรายงานทางการสำหรับพิมพ์/บันทึกเป็น PDF
- ดูข้อมูลที่ลบไปแล้วและกู้คืนได้

### เจ้าของสวนกาแฟ
- สมัครสมาชิกและเข้าสู่ระบบ
- แก้ไขข้อมูลบัญชีของตนเอง
- เพิ่ม แก้ไข ลบ และดูรายละเอียดสวนของตนเอง
- อัปโหลดรูปภาพ/สื่อประกอบ
- เพิ่ม แก้ไข ลบข้อมูลผลผลิต
- ดูรายงานสรุปผลผลิตของสวนตนเอง
- ส่งออกข้อมูลสวนของตนเองเป็น Excel, CSV และรายงานทางการสำหรับพิมพ์/บันทึกเป็น PDF
- ดูข้อมูลที่ลบไปแล้วและกู้คืนได้

### ผู้ใช้งานทั่วไป
- สมัครเพื่อสร้างบัญชีเจ้าของสวน
- ค้นหาสวนกาแฟตามพื้นที่หรือพันธุ์กาแฟ
- ดูตำแหน่งสวนกาแฟบนแผนที่
- ดูรายละเอียดข้อมูลกาแฟและข้อมูลพื้นฐานของแต่ละสวน

## วิธีติดตั้งฐานข้อมูล

1. เปิด XAMPP แล้ว Start MySQL
2. เข้า `http://localhost/phpmyadmin`
3. สร้างฐานข้อมูลชื่อ `coffee_loei_db`
4. Import ไฟล์ `database/coffee_loei_db.sql`

## วิธีรัน Backend

```bash
cd C:\xampp\htdocs\coffee_loei_db\backend
npm install
npm start
```

Backend จะรันที่:

```text
http://localhost:5000
```

## วิธีรัน Frontend

```bash
cd C:\xampp\htdocs\coffee_loei_db\frontend
npm install
npm run dev
```

Frontend จะรันที่:

```text
http://localhost:5173
```

## หมายเหตุ

- Navbar ถูกออกแบบใหม่ให้สั้นแบบเว็บทางการ โดยใช้เมนูหลักและ dropdown แทนการแสดงเมนูยาว
- React ใช้ Tailwind CSS แบบติดตั้งจริง ไม่ใช้ CDN
- หาก login ไม่ได้ ให้รันไฟล์ `database/reset_demo_passwords.sql` ใน phpMyAdmin

## อัปเดตเวอร์ชันล่าสุด
- Navbar จะแสดงเมนูตามบทบาทผู้ใช้ และจะไม่แสดงปุ่มเข้าสู่ระบบหลังจากเข้าสู่ระบบแล้ว
- ผู้ดูแลระบบจัดการข้อมูลได้ทุกส่วน: เจ้าของสวน สวนกาแฟ พันธุ์กาแฟ ประเภทสวน ประเภทดิน ผลผลิต รูปภาพ และข้อมูลที่ลบ
- เจ้าของสวนสามารถค้นหาดูข้อมูลทุกสวนได้ แต่จัดการได้เฉพาะสวน รูปภาพ และผลผลิตของตนเอง
- การส่งออกข้อมูลเปลี่ยนเป็นไฟล์ทางการ ได้แก่ Excel, CSV และรายงานสำหรับพิมพ์/บันทึกเป็น PDF
- กราฟจำนวนสวนใช้เลขจำนวนเต็ม เพราะหน่วยเป็นจำนวนแห่ง
- เพิ่มหน้าตั้งค่าบัญชีพื้นฐานสำหรับผู้ดูแลระบบและเจ้าของสวน

## อัปเดตระบบ State ล่าสุด

เวอร์ชันนี้ปรับระบบ state ของ React ใหม่ให้ใช้งานเนียนขึ้น:

- เพิ่ม `AuthContext` สำหรับเก็บสถานะผู้ใช้แบบ reactive
- หลังเข้าสู่ระบบ navbar จะเปลี่ยนทันที ไม่ต้องรีเฟรชหน้า
- หลังออกจากระบบเมนูจะกลับเป็นผู้ใช้ทั่วไปทันที
- Guard ของหน้า Admin/Owner อ่านสถานะจาก Context ไม่อ่านจาก localStorage ตรง ๆ
- เพิ่ม `AppStateContext` สำหรับ refresh ข้อมูลร่วมกันและ toast แจ้งเตือน
- ปรับ `api.js` ให้จัดการ 401 และสถานะ session อัตโนมัติ
- แก้การใช้งาน state ในหน้า Login, Register, Profile, Trash

ถ้าเปิดหน้า dev แล้วหน้าจอยังเป็นของเดิม ให้รัน:

```bash
cd frontend
npm run dev -- --force
```

หรือกด `Ctrl + Shift + R` หนึ่งครั้งเพื่อล้าง cache ของเบราว์เซอร์

## การปรับปรุง UI เวอร์ชันล่าสุด
- เพิ่มโครงสร้างหน้าเว็บแบบเป็นชั้นชัดเจน ได้แก่ Hero, บริการระบบ, โครงสร้างบทบาทผู้ใช้, Call to Action และ Footer
- เพิ่ม Footer แบบทางการ พร้อมเมนูหลัก ข้อมูลระบบ และข้อมูลติดต่อหน่วยงาน
- ปรับ Navbar ให้เป็นระเบียบ ใช้ Dropdown เพื่อลดความยาวของเมนู
- ปรับหน้าแรกให้มีเลเยอร์ ภาพประกอบ การ์ด และอนิเมชันที่ทันสมัยมากขึ้น
